## CSV 한글이 `?`로 바뀌는 이유

- **Excel**: "다른 이름으로 저장" → "CSV (쉼표 구분)"으로 저장하면 **시스템 기본 인코딩**(예: CP949)으로 저장됩니다.  
  한글이 들어 있는 파일을 UTF-8이 아닌 인코딩으로 저장하면 글자가 깨지거나 `?`로 바뀝니다.
- **해결**: Excel에서 저장할 때 **"CSV UTF-8(쉼표 구분)"**을 선택하세요.
- **편집기**: VS Code / Cursor에서는 하단 상태栏의 인코딩을 **UTF-8**로 두고 저장하세요.

## 한글이 이미 `?`로 깨진 경우

프로젝트 루트에서 다음을 실행하면 `key_expressions_grade3.csv`를 UTF-8로 다시 저장하고,  
첫 번째 데이터 행의 한글과 출판사명을 복구합니다.

```bash
python fix_csv_encoding.py
```

다른 단원/출판사 데이터는 CSV를 **항상 UTF-8(또는 UTF-8 BOM)**으로 저장한 채로 편집하면 한글이 유지됩니다.
